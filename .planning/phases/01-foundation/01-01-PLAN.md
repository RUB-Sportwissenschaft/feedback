**CSS design system (replicate pruefung exactly):**
- All CSS variables from the interfaces block above (--orange, --blau, --bg, etc.)
- Reset: `*, *::before, *::after { box-sizing: border-box; margin: 0; padding: 0; }`
- Body: font-family RubFlama with system-ui fallback, background --bg, color --text, min-height 100vh, flex column
- `.form-section` card pattern (from interfaces)
- `.card-grid`, `.card`, `.card.selected`, `.card-snowboard` tile patterns (from interfaces)
- `main` layout: max-width 560px, centered, padding 1.5rem 1rem 3rem

**Header:**
- Blue background (--blau), white text, padding 0.75rem 1.25rem, flex with gap
- h1: "Lehrgangskritik Valmorel", font-size 1.15rem, font-weight 700
- Optional ski emoji as header-icon (use `&#x26F7;` HTML entity)
- NO orange accent bar below the header (per user decision)

**Sticky progress indicator (directly below header):**
- `position: sticky; top: 0; z-index: 100;` (header is NOT sticky, only the progress bar)
- Background: --white with bottom border --border and shadow-sm
- Layout: A progress bar track (full width, 4px height, --bg background) with an orange fill div (--orange), plus a step counter on the right ("1 / 7")
- Below the bar: current section name in --text-muted, smaller font
- Padding: 0.6rem 1rem
- The progress bar fill width should be computed as `(currentStep / totalSteps) * 100%` via a CSS variable `--progress` set by JS

**Step container structure:**
- Inside `<main>`, create 7 `<div>` elements with class `step` and data-step="0" through data-step="6"
- Only the active step is visible (`display: block`), all others hidden (`display: none`)
- Step 0: "Gruppe &amp; Name" (built in Task 2)
- Steps 1-6: empty placeholder divs (filled by Plan 02)

**Navigation buttons:**
- A fixed bottom bar OR buttons at the bottom of each step
- "Zur\u00fcck" button (left, ghost/outline style: transparent background, --blau border and text) and "Weiter" button (right, filled: --orange background, white text)
- Step 0: only "Weiter" (no Zur\u00fcck). Steps 1-5: both. Step 6: "Zur\u00fcck" + "Absenden" (submit button, styled differently -- but for Phase 1 skeleton just use "Weiter" placeholder)
- Buttons min-height 44px for touch targets
- Place navigation inside each step div at the bottom (not a global fixed bar) so each step controls its own buttons

**Wizard JS (minimal, at end of body in `<script>`):**
- `let currentStep = 0; const totalSteps = 7;`
- `function showStep(n)`: hides all `.step` divs, shows the one with `data-step=n`, updates progress bar fill width and step counter text, updates section name
- Section names array: `['Gruppe & Name', 'S1 \u2014 Rahmenbedingungen', 'S3 \u2014 Organisation', 'S2 \u2014 Ausbildung', 'S4 \u2014 Pr\u00fcfung', 'S5 \u2014 Soziales', 'S6 \u2014 Ausbilder & Abschluss']`
- `function nextStep()` / `function prevStep()`: increment/decrement currentStep, call showStep
- Wire Weiter/Zur\u00fcck buttons via onclick
- Call `showStep(0)` on DOMContentLoaded

**IMPORTANT per user decisions:**
- NO orange accent bar below header
- Progress bar fill color is --orange (#EC633A)
- Multi-step wizard, NOT single scrollable page
